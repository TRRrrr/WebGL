Class Detail
okTexture(Context)
Texturing maps a portion of one or more specified images onto each primitive for which texturing is enabled. It also can be used as a render target to receive the framebuffer output.
Parameters:
{WebGLContext} Context
The webgl context of the target canvas.
Returns:
{okTexture} A new okTexture object
Method Detail
{Null} bind(TexPos)
Bind this texture to the WebGL context as the current active texture.
Parameters:
{Number} TexPos Optional
Indicates binding to which active texture position. The default value is 0.
Deprecated:
Using this method to bind texture directly to WebGL context is unrecommended. A better method is to use okProgram#setTexture and okProgram#setTextureArray.
Returns:
{Null}
{Null} createTexture(TexType, Width, Height, Format, DataType, DataX, DataNX, DataY, DataNY, DataZ, DataNZ)
Initialize the texture.
Parameters:
{Enum} TexType
The type of the texture. The available enumeration is [TEXTURE_2D, TEXTURE_CUBE_MAP].
{Number} Width
The width of the texture, just give -1 if the data source is a HTML element cause the size will be read from the element.
{Number} Height
The height of the texture, just give -1 if the data source is a HTML element cause the size will be read from the element.
{Enum} Format
The pixel format of the texture. The available enumeration is [ALPHA, RGB, RGBA].
{Enum} DataType
The basic data type of the pixel of the texture. The available enumeration is [UNSIGNED_BYTE, UNSIGNED_SHORT_5_6_5, UNSIGNED_SHORT_4_4_4_4, UNSIGNED_SHORT_5_5_5_1].
{Image | Array} DataX Optional
The image to initialize the texture. | The pixel data array to initialize the texture. If this is a cubemap texture, then this parameter is used for the positive-X surface of the cube. If this parameter is ignored, then the texture will be initialized to an empty texture.
