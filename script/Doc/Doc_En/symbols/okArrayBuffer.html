okArrayBuffer(Context)
OpenGLES buffer objects provide a mechanism that clients(CPU side) can use to allocate, initialize and render from memory. Buffer objects can be used to store vertex array and element index data.
Parameters:
{WebGLContext} Context
The webgl context of the target canvas.
Returns:
{okArrayBuffer} A new okArrayBuffer object

{Null} bindAttrib(AttribIdx, ElementCount, Offset, Stride)
Bind this array buffer object to the WebGL context as a shader attribute source.
Parameters:
{Number} AttribIdx
Specifies the index of the generic vertex attribute to be modified.
{Number} ElementCount
Indicate a vertex needs how many base data elements. For example, if the data content of the buffer is used as vertex position <x, y, z>, then the position of one vertex is composed by 3 float data element.
{Number} Offset Optional
Specifies the element offset to the first component of the first vertex attribute in the array. The default value is 0.
{Number} Stride Optional
Specifies the element offset between consecutive vertex attributes. If stride is 0, the generic vertex attributes are understood to be tightly packed in the array. The default value is 0.
Returns:
{Null}
Requires:
This is a vertex array buffer(be initialized with the type ARRAY_BUFFER).
{Null} createBuffer(BufType, DataType, Usage, DataPara)
Initialize the buffer. This method is used to initialize the buffer object. Users can use this method initializing a buffer with initial data or an empty buffer with a specified length.
Parameters:
{Enum} BufType
Indicates the buffer object is used to store vertex array or index data. The available enumeration is [ARRAY_BUFFER, ELEMENT_ARRAY_BUFFER].
{Enum} DataType
The basic data type of the array. If this is a vertex array buffer, then the data type should be FLOAT. If this is a index array buffer, then the data type should be UNSIGNED_SHORT.
{Enum} Usage Optional
The usage of this buffer. The available enumeration is [STATIC_DRAW, DYNAMIC_DRAW, STREAM_DRAW]. The default value is STATIC_DRAW.
