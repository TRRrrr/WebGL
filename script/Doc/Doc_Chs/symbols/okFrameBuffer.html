Class okFrameBuffer

帧缓冲类。

Class Summary
Constructor Attributes Constructor Name and Description
 
okFrameBuffer(Context)
通过Frame Buffer, 开发者可自定义渲染管线(Render Pipeline)中的深度缓存(Depth Buffer), 模板缓存(Stencil Buffer)以及渲染目标(Render Target)。
纹理和渲染缓存（renderbuffer）可以绑定在Frame Buffer上储存渲染输出。
Method Summary
Method Attributes Method Name and Description
 
attachDepthBuffer(指定绑定的Render)
绑定一个okRenderBuffer对象到此Frame Buffer上作为Depth Buffer。
注意, 执行此方法前必须先执行bind方法将此Frame Buffer作为WebGL上下文的当前Frame Buffer。
 
attachDepthStencilBuffer(指定绑定的Render)
绑定一个okRenderBuffer对象到此Frame Buffer上作为DepthStencil Buffer。
注意, 执行此方法前必须先执行bind方法将此Frame Buffer作为WebGL上下文的当前Frame Buffer。
 
attachRenderTexture(指定绑定到当前FrameBuffer的第几个渲染目标。当前阶段，只有“0”是可用值。, 指定要绑定的Texture)
绑定一个okTexture对象到此Frame Buffer的指定渲染目标上，作为颜色输出。
注意， 执行此方法前必须先执行bind方法将此Frame Buffer作为WebGL上下文的当前Frame Buffer。
 
attachStencilBuffer(指定绑定的Render)
绑定一个okRenderBuffer对象到此Frame Buffer上作为Stencil Buffer。
注意, 执行此方法前必须先执行bind方法将此Frame Buffer作为WebGL上下文的当前Frame Buffer。
 
bind()
将此Frame Buffer作为WebGL上下文的当前Frame Buffer.
 
createBuffer()
初始化WebGL Frame Buffer对象。
 
releaseBuffer()
释放此WebGL Frame Buffer对象。 这个方法将会释放由此Frame Buffer生成的WebGL资源。
在调用本方法之后，此Frame Buffer将不可用，直至下一次被重新初始化。
 
unbind()
将默认Frame Buffer作为WebGL上下文的当前Frame Buffer。 本方法必须和bind方法配对使用。
