Class okResourceManager

Oak3D Engine的资源管理器。

Class Summary
Constructor Attributes Constructor Name and Description
<private> <inner>  
okResourceManager(engineSetting, renderContext)
okResourceManager为Oak3D Engine的资源管理器，负责模型、纹理、动作等资源的IO与管理。
用户不可以直接创建okResourceManager对象，它是作为okEngine对象的内部成员提供给开发者，可直接从已经成功初始化的okEngine对象中获取。
一般情况下，开发者不必直接操纵资源管理器，资源加载操作会在场景物件内部完成。若需要直接使用okResourceManager加载资源，需要了解一下okResourceManager的资源管理逻辑。okResourceManager依赖关键字管理资源引用，因此对于相同关键字的资源加载请求，管理器内部会直接返回已存在的同关键字资源，而不会重新创建。场景物件默认加载资源，使用资源的文件引用地址作为关键字，因此若手动加载的资源需要与自动加载逻辑配合， 也应遵守这一规则。
